# 기존 `Skeleton3D.tsx` 코드를 유지하기 위한 JSON 데이터 제공 가이드

사용자님의 기존 아바타 연결 코드(`Skeleton3D.tsx`)가 이미 매우 잘 동작하고 있으므로, **코드를 수정하지 않고 외부 JSON 데이터의 형태를 코드에 맞추는 방법**을 안내해 드립니다.

현재 코드는 **MediaPipe Pose** 모델의 출력 형식(33개의 랜드마크 배열)을 그대로 부어주면 알아서 동작하도록 설계되어 있습니다. 
따라서, JSON 파일은 복잡한 쿼터니언 회전값 없이 오로지 **특정 인덱스 순서에 맞는 `x, y, z` 3D 점 좌표(Position) 배열**만 제공하면 완벽하게 호환됩니다.

---

## 1. 필요한 JSON 데이터 구조

기존 코드에서 사용하는 `landmarks` 속성에 전달될 수 있도록, 파일 또는 웹소켓을 통해 아래와 같은 **배열(Array)** 구조를 전달받아야 합니다.

### 단일 프레임(1 Frame) 기준 데이터 포맷
배열안에 `{ x: 소수점, y: 소수점, z: 소수점 }` 형태의 객체가 **최소 29개(인덱스 0~28)** 순서대로 들어있어야 합니다.

> **💡 앞뒤(Depth) 구분 기능**
> 객체 안의 **`z` 값**이 바로 앞뒤(깊이)를 나타내며 이를 통해 3D 입체성을 가집니다. (이 값이 존재해야 팔을 앞으로 뻗었는지, 뒤로 뻗었는지 구분이 가능합니다.)
> - **`x`**: 좌우 위치
> - **`y`**: 상하 위치
> - **`z`**: 앞뒤 위치 (보통 골반 기준점을 0으로 잡고, 카메라와 가까워지면 음수(-), 멀어지면 양수(+) 값을 가집니다.)


```json
[
  { "x": 0.0, "y": 0.0, "z": 0.0 }, // [0] 코 (사용 안 함)
  { "x": 0.0, "y": 0.0, "z": 0.0 }, // [1] 눈 (사용 안 함)
  // ... (0번부터 10번까지는 코드에서 사용하지 않으므로 0으로 채워도 무방함) ...
  { "x": 0.0, "y": 0.0, "z": 0.0 }, // [10] 입 (사용 안 함)
  
  // ▼ 양팔 데이터
  { "x": 0.15, "y": 0.50, "z": -0.1 }, // [11] 왼쪽 어깨 (Left Shoulder)
  { "x": -0.15, "y": 0.50, "z": -0.1 },// [12] 오른쪽 어깨 (Right Shoulder)
  { "x": 0.20, "y": 0.30, "z": -0.2 }, // [13] 왼쪽 팔꿈치 (Left Elbow)
  { "x": -0.20, "y": 0.30, "z": -0.2 },// [14] 오른쪽 팔꿈치 (Right Elbow)
  { "x": 0.25, "y": 0.10, "z": -0.3 }, // [15] 왼쪽 손목 (Left Wrist)
  { "x": -0.25, "y": 0.10, "z": -0.3 },// [16] 오른쪽 손목 (Right Wrist)
  
  // ... (17 ~ 22번 손가락 관련 랜드마크. 사용 안 함) ...
  { "x": 0.0, "y": 0.0, "z": 0.0 },
  { "x": 0.0, "y": 0.0, "z": 0.0 },
  { "x": 0.0, "y": 0.0, "z": 0.0 },
  { "x": 0.0, "y": 0.0, "z": 0.0 },
  { "x": 0.0, "y": 0.0, "z": 0.0 },
  { "x": 0.0, "y": 0.0, "z": 0.0 },

  // ▼ 몸통 및 양다리 데이터
  { "x": 0.10, "y": -0.20, "z": 0.0 }, // [23] 왼쪽 골반 (Left Hip)
  { "x": -0.10, "y": -0.20, "z": 0.0 },// [24] 오른쪽 골반 (Right Hip)
  { "x": 0.10, "y": -0.50, "z": 0.1 }, // [25] 왼쪽 무릎 (Left Knee)
  { "x": -0.10, "y": -0.50, "z": 0.1 },// [26] 오른쪽 무릎 (Right Knee)
  { "x": 0.12, "y": -0.80, "z": 0.2 }, // [27] 왼쪽 발목 (Left Ankle)
  { "x": -0.12, "y": -0.80, "z": 0.2 } // [28] 오른쪽 발목 (Right Ankle)
]
```

### 💡 (팁) 배열을 만들 때 주의사항
* 현재 아바타 코드는 `11~16`번(팔)과 `23~28`번(몸통/다리) 인덱스만 사용합니다.
* 배열 길이는 꼭 33개가 아니어도 되지만, **사용하는 인덱스의 번호(위치)**는 정확히 맞아야 합니다. 따라서 최소 `29` 칸짜리 배열이어야 하며, 안 쓰는 부위는 `{ "x": 0, "y": 0, "z": 0 }` 형태의 더미(Dummy) 데이터를 넣어 배열 길이를 맞춰 주셔야 에러가 나지 않습니다.

## 2. 서버/API 전체 응답 포맷 (권장)

만약 동영상 애니메이션을 위해 1프레임 단위가 아닌 전체 영상(예: 30프레임) 단위로 JSON을 받는다면, 다음과 같이 프레임 단위의 랜드마크 배열을 `List` 로 묶어서 보내주면 됩니다.

```json
{
  "fps": 30,
  "total_frames": 100,
  "animation_data": [
    // 0번째 프레임의 랜드마크 배열
    {
      "frame": 0,
      "landmarks": [
        { "x": 0.0, "y": 0.0, "z": 0.0 }, // [0]
        // ... (1번 ~ 10번) ...
        { "x": 0.1, "y": 0.5, "z": 0.0 }, // [11] 왼쪽 어깨
        // ... (이하 생략) ...
      ]
    },
    // 1번째 프레임의 랜드마크 배열
    {
      "frame": 1,
      "landmarks": [ ... ]
    }
  ]
}
```

## 3. 요약 (서버/데이터 제작자에게 요청할 사항)

현재 로직을 그대로 사용하고 싶다면, `serve_threejs_data.json` 포맷을 생성하는 담당자(또는 데이터 전처리 스크립트)에게 아래의 내용을 그대로 요청하시면 됩니다.

> **"복잡한 로컬/월드 회전값과 계층형 객체 구조는 필요 없습니다. 각 프레임마다 MediaPipe Pose 구조와 완전히 동일하게 0번부터 28번까지의 X, Y, Z 위치값(Position)만 있는 단순 배열(`List<{x, y, z}>`)을 만들어서 제공해 주세요."**
